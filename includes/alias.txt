=== Générique utils ===
www     : cd /var/www/vhosts
cl      : clear

=== Plesk utils ===
rep        : Lancement de la commande `plesk-repair` sur l’instance en cours
repall     : Lancement de la commande `plesk-repair` sur l’ensemble des instances
getplesksite : Récupération du nom de  l’instance en cours

=== Accès aux fichiers ===
bashrc  : Accéder au fichier `.bashrc` via `nano`
mbashrc : Accéder au fichier `.bashrc` via `micro`
saverc  : Sauvegarder les modifications faites au fichier `.bashrc`
maliass : Éditer ce fichier de liste des alias
aliass  : Alias qui affiche uniquement les alias que l’on a créés
alias   : Commande qui affiche nos alias (sans les fonctions) ainsi que les alias UNIX déjà existants

=== Commandes spécifiques client ===
impots : cd /var/www/vhost/sandbox.snpnc.org/httpdocs/snpnc-impots
cnec   : envoi du rapport pour la création des comptes web pour le CNEC (via Make)

=== Manipulations globales d'instances ===
backup   : Sauvegarde une instance
clone    : Lancement du script de clonage d’instances
clonebdd : Lancement du script de clonage de base de données
remp     : Lancement du script de remplacement de valeurs dans la base de données
up       : Lancement du script civi-update.sh, pour la mise à jour des versions CiviCRM de l’instance souhaitée
vcv      : Affichage de toutes les instances avec leur version de CiviCRM et PHP
vcms     : Affichage de toutes les instances avec leur version de CMS et PHP

=== Manipulations de base de données ===
getbdd : Récupère les identifiants de la base de données de l’instance courante
sql    : Ouvre le terminal SQL de l’instance courante, peut être utilisé sous la forme `sql < fichier.sql`
dump   : Récupère l’intégralité de la base de données ; si aucune instruction complémentaire n’est donnée (ex : dump > dest.sql), l’affiche dans le terminal

=== Lister les extensions CiviCRM ===
list   : Lister toutes les extensions présentes
listi  : Lister les extensions installées
listd  : Lister les extensions désinstallées
listu  : Lister les extensions supprimées
listup : Lister les extensions installées et pouvant être mises à jour

=== Gérer les extensions CiviCRM d’une instance ===
en       : Activation d’une ou plusieurs extensions CiviCRM déjà présentes dans l’instance
dll      : Installation et activation d’une ou plusieurs extensions CiviCRM
dis      : Désactivation d’une ou plusieurs extensions CiviCRM déjà présentes dans l’instance
un       : Désinstallation d’une ou plusieurs extensions, avec suppression du dossier correspondant dans le dossier `extensions`
cvgc     : `git clone` + activation de l’extension CiviCRM
cvgp     : `git pull` d’une extension CiviCRM déjà clonée
cvextup  : Met à jour le code de toutes les extensions CiviCRM pour une nouvelle version de civix
cvallup  : Met à jour toutes les extensions de CiviCRM
allup    : Alias de `cvallup`

=== Gérer une instance CiviCRM ===
cvup     : Mise à jour de la BDD CiviCRM de l’instance
cvf      : Vidage simple du cache de CiviCRM
cvff     : Flush complet — vidage du cache de CiviCRM + reconstruction des déclencheurs ; plus lent mais plus profond que `cvf`
cvpatch  : Patch du core de CiviCRM, depuis GitHub
cvdall   : Désactive toutes les extensions CiviCRM
cvdall --cancel : annule la dernière opération de cvdall dans l'instance
cvall    : Active toutes les extensions CiviCRM
cvall --cancel : annule la dernière opération de cvall dans l'instance

=== Gérer une instance WordPress ===
wpf     : Vider le cache de WordPress
wpdeb   : Activer le mode debug
wpudeb  : Désactiver le mode debug

=== Gérer les extensions WordPress d’une instance ===
wpgc   : `git clone` + activation de l’extension WordPress
wpgp   : `git pull` d’une extension WordPress
wpi    : Installer une extension WordPress
wpa    : Activer une extension WordPress
wpd    : Désactiver une extension WordPress
wpu    : Désinstaller une extension WordPress
wpup   : Mettre à jour une extension WordPress
wpall  : Activer toutes les extensions WordPress
wpall --cancel : Annuler l’activation de toutes les extensions WordPress
wpdall : Désactiver toutes les extensions WordPress
wpdall --cancel : Annuler la désactivation de toutes les extensions WordPress

=== Gérer une instance Backdrop ===
beef   : Vider le cache de Backdrop

=== Gérer une instance Drupal ===
dm     : mettre en maintenance
dum    : retirer la maintenance
